Details

MountOperation provides a mechanism for interacting with the user. It can be used for authenticating mountable operations, such as loop mounting files, hard drive partitions or server locations. It can also be used to ask the user questions or show a list of applications preventing unmount or eject operations from completing.

Note that MountOperation is used for more than just Mount objects – for example it is also used in driveStart.

Users should instantiate a subclass of this that implements all the various callbacks to show the required dialogs, such as MountOperation. If no user interaction is desired (for example when automounting filesystems at login time), usually Nothing can be passed, see each method taking a MountOperation for details.

Signals

mountOperationAborted :: MountOperationClass op => Signal op (IO ())

Emitted by the backend when e.g. a device becomes unavailable while a mount operation is in progress.

Implementations of MountOperation should handle this signal by dismissing open password dialogs.
